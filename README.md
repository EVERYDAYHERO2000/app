# Bulk Cargo PWA

Каркас PWA для сценариев клиент / админ / водитель (детальная спецификация при необходимости хранится локально и не входит в репозиторий).

## Структура

- **backend** — Node.js, TypeScript, Express, Prisma, SQLite
- **frontend** — React, TypeScript, Material UI, PWA (Vite)
- **mvp** — сохранённая версия текущего MVP (HTML/JS)
- **docs** — технические заметки

## Быстрый старт

### Локальная разработка без Docker (основной сценарий)

0. Node.js:
   - проект настроен под текущую версию в окружении: `v17.9.1` (см. `.nvmrc`);
   - при использовании `nvm`: `nvm use`.

1. Скопировать env backend:
   ```bash
   cp backend/.env.example backend/.env
   ```

2. Выполнить первичную настройку одной командой:
   ```bash
   npm run setup
   ```

3. Запуск:
   - Backend: `npm run dev:backend`
   - Frontend: `npm run dev:frontend`

Фронт: http://localhost:5173. API проксируется на http://localhost:3001 через `/api`.

Для локальной авторизации в dev фронт автоматически добавляет `x-dev-user-id` из `frontend/.env`.
По умолчанию после seed используются стабильные UUID:
- CLIENT: `11111111-1111-1111-1111-111111111101`
- ADMIN: `11111111-1111-1111-1111-111111111102`
- DRIVER: `11111111-1111-1111-1111-111111111103`

### Docker (опционально)

Docker для локальной разработки больше не нужен. Для старта используйте:

```bash
npm run setup
```

## Основное по спецификации

- Экраны клиента: авторизация, профиль, список заказов, детали заказа, создание заказа.
- Админка: все заказы, смена статусов, назначение водителя; материалы и подматериалы с ценами.
- Водитель: назначенные заказы, детали, маршрут.
- Статусы заказа: NEW → ACCEPTED → CONFIRMED/REJECTED → IN_PROGRESS → COMPLETED.
- База: SQLite, таблицы users, orders, order_status_history, materials, submaterials.

Подробный чеклист требований при необходимости ведётся вне репозитория (локальные `AI_*.md` в `.gitignore`).
